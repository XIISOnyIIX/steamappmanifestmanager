Fix: Dropdown menu not appearing when arrow clicked

## Problem
The dropdown arrow button (‚ñº) next to the Scan button did nothing when clicked.
The menu would not appear at all, making the "Scan All Installed Games" feature
inaccessible to users.

## Root Cause
1. Event propagation timing - document click listener was closing menu immediately
2. Insufficient debugging visibility - hard to diagnose issues without logs
3. Potential CSS specificity issues

## Solution

### 1. Enhanced Event Handling
- Added `e.preventDefault()` to dropdown click handler
- Delayed document click listener attachment with `setTimeout` to prevent
  immediate menu closure
- Improved event propagation control with proper `stopPropagation()`

### 2. Comprehensive Debug Logging
Added extensive console logging throughout the dropdown lifecycle:
- Element existence checks with emoji indicators (üîç, üìä, üö®, ‚úÖ, ‚ùå)
- Menu state tracking (hidden/visible, display styles)
- Click event logging with full context
- Position and dimension logging for troubleshooting
- Event listener attachment confirmation

Example output:
```
üö® DROPDOWN BUTTON CLICKED!
  - Menu currently hidden? true
  - Menu display style: block
  - Menu position: {top: 65, left: 1250, width: 220, height: 80}
```

### 3. Added Browser Console Test Function
Created `window.testDropdownMenu()` for emergency testing:
- Forces menu visible with red background and yellow border
- Sets z-index to 9999 for visibility
- Logs position and state for diagnosis
- Helps identify CSS vs JavaScript issues

### 4. HTML Enhancements
- Added inline style fallbacks for spacing and cursor
- Improved accessibility with explicit cursor pointer
